# Exercícios - Semana 9

Esta pasta contém os materiais práticos utilizados durante a **Semana 9**, com foco em introdução e prática de SQL aplicado a dados trabalhados em aula.

O objetivo principal desta semana é revisar o fluxo de leitura e preparação de dados, iniciar consultas SQL e praticar os principais comandos vistos nas aulas.

---

## Arquivos disponíveis

### `SEMANA_9.ipynb`

Notebook com **somente os exemplos e exercícios das aulas 1, 2 e 3 da Semana 9**.

Este arquivo deve ser usado como material principal de acompanhamento das aulas da semana.

Conteúdos abordados:

- Diferença entre Python e SQL;
- Introdução ao SQL;
- `SELECT`;
- `WHERE`;
- `ORDER BY`;
- `JOIN`;
- `GROUP BY`;
- `SUM`;
- `COUNT`;
- `AVG`;
- `HAVING`;
- Funções de data, como `DATE_TRUNC` e `EXTRACT`.

---

### `Semana9_Projeto_SQL_Aluno_Completo.ipynb`

Notebook mais completo, com exercícios relacionados a:

- Leitura de arquivos originais;
- Entendimento inicial dos dados;
- Limpeza e preparação dos dados;
- Organização dos dados para análise;
- Primeiros comandos em SQL.

Este arquivo é mais extenso e trabalha conteúdos da Semana 8 até iniciarmos SQL


---

### `Semana_9_Lista_Exercicios_SQLBásico.ipynb`

Lista de exercícios para aplicar os conceitos de SQL utilizando os dados trabalhados durante a semana.

Este notebook deve ser usado para prática individual, revisão e consolidação dos conteúdos vistos em aula.

Conteúdos praticados:

- Seleção de colunas;
- Filtros;
- Ordenação;
- Junção de tabelas;
- Agrupamentos;
- Cálculos com agregações;
- Filtros em dados agrupados.

---

---

### `Semana_9_Exercicios_Opcionais_SQL.ipynb`

Notebook com **exercícios opcionais** para reforço dos conceitos de SQL.

Os dados utilizados neste arquivo são **fictícios e criados dentro do próprio notebook**, ou seja, não é necessário baixar bases externas.

Objetivo do arquivo:

- Fixar os conceitos ensinados em aula;
- Praticar comandos SQL em exemplos mais simples;
- Reforçar a lógica de consultas;
- Permitir treino extra para quem quiser revisar o conteúdo.


## Orientações para os alunos

1. Faça o **fork** do repositório para a sua conta do GitHub.
2. Trabalhe sempre no seu próprio fork.
3. Não altere diretamente o repositório original da professora.
4. Faça os exercícios no seu notebook.
5. Salve as alterações.
6. Faça o commit das suas alterações no seu repositório.
7. Envie o push para o seu GitHub.

---

## Sugestão de ordem de estudo

A ordem recomendada para usar os arquivos é:

1. `SEMANA_9.ipynb`
2. `Semana_9_Lista_Exercicios_SQLBásico.ipynb`
3. `Semana_9_Exercicios_Opcionais_SQL.ipynb`
